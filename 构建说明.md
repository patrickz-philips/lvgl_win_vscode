# LVGL 项目构建说明（简体中文）

## 📋 快速开始

### 构建项目
```batch
# 构建 Hair Dryer - Debug模式（默认）
build.bat

# 构建 Hair Dryer - Release模式
build.bat HAIR_DRYER Release

# 构建 Smart Shaver - Debug模式
build.bat SMART_SHAVER Debug
```

### 运行项目
```batch
# 运行 Hair Dryer - Debug版本
run.bat

# 运行 Hair Dryer - Release版本
run.bat HAIR_DRYER Release
```

### 清理项目
```batch
# 清理所有构建文件
clean.bat

# 仅清理某个项目
clean.bat HAIR_DRYER
```

## 🎯 主要改进

### ✅ 已完成的优化

1. **清晰的目录结构**
   - 构建文件: `build/{Debug|Release}/{project}/`
   - 输出文件: `bin/{Debug|Release}/{project}/`
   - 不同项目和构建类型完全隔离

2. **统一的构建脚本**
   - `build.bat` - 构建任意项目和配置
   - `run.bat` - 运行指定项目
   - `clean.bat` - 灵活的清理选项

3. **只编译需要的库**
   - ❌ 不编译 LVGL Examples
   - ❌ 不编译 LVGL Demos  
   - ❌ 不编译 ThorVG
   - ✅ 只编译核心LVGL库
   - **结果**: 更快的编译速度！

4. **灵活的构建选择**
   - 选择项目: HAIR_DRYER 或 SMART_SHAVER
   - 选择模式: Debug 或 Release
   - 独立构建，互不干扰

## 📊 Debug vs Release 对比

| 特性 | Debug | Release |
|------|-------|---------|
| 编译速度 | ⚡ 快 | 较慢 |
| 运行速度 | 正常 | ⚡ 更快 |
| 文件大小 | 大（含调试符号） | 💾 小 |
| 资源加载 | 动态加载文件 | 静态编译到exe |
| 调试支持 | ✅ 完整 | ❌ 有限 |

**建议**:
- 🔧 开发时用 Debug - 快速编译，方便调试
- 🚀 发布时用 Release - 优化性能，减小体积

## 🛠️ 脚本参数说明

### build.bat
```batch
build.bat [项目名] [构建类型]
```
- **项目名**: HAIR_DRYER（默认）、SMART_SHAVER
- **构建类型**: Debug（默认）、Release

### run.bat
```batch
run.bat [项目名] [构建类型]
```
- 参数同 build.bat

### clean.bat
```batch
clean.bat [目标]
```
- **all** - 清理所有（默认）
- **build** - 仅清理构建文件
- **bin** - 仅清理输出文件
- **HAIR_DRYER** - 仅清理该项目
- **SMART_SHAVER** - 仅清理该项目

## 📝 使用示例

### 场景1: 开发 Hair Dryer 项目
```batch
# 第一次构建
build.bat HAIR_DRYER Debug

# 修改代码后重新构建
build.bat HAIR_DRYER Debug

# 运行测试
run.bat HAIR_DRYER Debug
```

### 场景2: 准备发布版本
```batch
# 构建Release版本
build.bat HAIR_DRYER Release

# 测试Release版本
run.bat HAIR_DRYER Release

# 发布文件在: bin\Release\hair_dryer\
```

### 场景3: 切换到另一个项目
```batch
# 构建Smart Shaver
build.bat SMART_SHAVER Debug

# 运行Smart Shaver
run.bat SMART_SHAVER Debug

# 注意：不影响之前的Hair Dryer构建！
```

### 场景4: 清理重新开始
```batch
# 遇到问题时，清理所有
clean.bat all

# 重新构建
build.bat
```

## ⚠️ 注意事项

1. **旧脚本**: `build_debug.bat` 和 `build_release.bat` 仍然可用，但已标记为过时，建议使用新的 `build.bat`

2. **Hair Dryer Release模式**: 需要 Python 和 Pillow 来转换图片
   ```batch
   pip install pillow
   ```

3. **SDL2依赖**: 确保已安装（通过vcpkg）
   ```batch
   vcpkg install sdl2:x64-windows
   ```

## 🔍 故障排除

### 问题：找不到可执行文件
**检查**:
1. 构建是否成功？
2. 路径是否正确: `bin\{Debug|Release}\{project}\main.exe`

### 问题：编译错误
**解决**:
```batch
# 1. 清理所有
clean.bat all

# 2. 重新构建
build.bat [项目] [模式]
```

### 问题：SDL2错误
**解决**:
```batch
vcpkg install sdl2:x64-windows
vcpkg integrate install
```
